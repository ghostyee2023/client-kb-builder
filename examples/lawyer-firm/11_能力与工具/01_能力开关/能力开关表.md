# 能力开关表

| capability_id | 能力名称 | 类型 | 状态 | 服务节点 |
| --- | --- | --- | --- | --- |
| `matter_intake_classifier` | 咨询来源与案由识别 | template | on | 新咨询收件 |
| `conflict_check` | 冲突检查 | rule | core | 冲突与风险初筛 |
| `confidentiality_governance` | 保密与权限治理 | rule | core | 全流程 |
| `evidence_catalog_builder` | 证据目录整理 | template | on | 证据整理 |
| `legal_work_product_template` | 法律工作产品模板 | template | on | 方案、文书、交付 |
| `ai_consultation_summary` | 咨询摘要辅助 | ai_assist | manual | 咨询摘要、访谈整理 |
| `legal_research_digest` | 法律检索摘要辅助 | ai_assist | manual | 法律检索摘要 |
| `client_progress_brief` | 客户进度同步草稿 | template | manual | 办案交付 |
| `case_review_extractor` | 单案复盘抽取 | template | manual | 复盘沉淀 |

## 规则

- `core` 能力不能关闭。
- `manual` 能力必须由律师明确触发。
- AI 辅助能力不能直接生成最终法律意见。
- 任何自动化上线前，必须先定义读取范围、写入范围和失败回退。
