# skill安装与启用清单

## 安装前检查

| 检查项                            | 状态      | 说明                                     |
| ------------------------------ | ------- | -------------------------------------- |
| 客户使用 Codex / Claude / 其他 Agent | pending | 决定 `AGENTS.md`、`CLAUDE.md` 或其他规则文件是否生效 |
| 系统级 skill 候选清单                 | pending | 来自交付方本机可同步清单                           |
| 是否允许项目级 skill                  | pending | 由客户工具环境决定                              |
| 是否允许读取客户材料                     | pending | 必须先定义权限和保密规则                           |
| 是否允许写入案件文件                     | pending | 必须先定义写入范围                              |
| 是否需要本地脚本                       | pending | 首版默认不用脚本                               |

## 启用顺序

1. 启用根目录执行规则：`AGENTS.md` / `CLAUDE.md`。
2. 启用业务流开关：`10_治理规则/业务流开关表.md`。
3. 启用能力开关：`11_能力与工具/01_能力开关/能力开关表.md`。
4. 选择系统级 skill：`11_能力与工具/01_能力开关/系统级skill候选清单.md`。
5. 登记项目级 skill：`11_能力与工具/01_能力开关/项目级skill清单.md`。
6. 单个节点试跑：新咨询记录、冲突检查、证据目录。
7. 写验证记录：`11_能力与工具/07_验证记录/`。

## 首版建议

先安装或准备最小系统级 skill 候选：

- `documents:documents`
- `pdf:pdf`
- `spreadsheets:Spreadsheets`

项目级 skill 先不真实开发，只用模板和 AI 执行规则试跑。

当客户连续跑通 3 个真实咨询后，再把高频节点封装成项目级 skill：

- 新咨询结构化。
- 证据目录整理。
- 客户进度同步。
- 单案复盘抽取。

## 禁止事项

- 不允许自动安装未登记 skill。
- 不允许 skill 直接读取全部客户材料。
- 不允许 skill 绕过律师确认生成最终法律意见。
- 不允许把客户私密案件内容写入公共方法库。
