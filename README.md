# client-kb-builder

`client-kb-builder` 是一个面向客户知识库交付场景的项目级 skill。

它帮助创作者、顾问、AI 转型服务商和知识库搭建者，把一个客户的真实业务，从模糊需求一步步翻译成可运行、可配置、可交付、可复盘的知识库操作系统。

这个项目不是一套固定目录模板，也不是把某个现成知识库复制给客户。它更像一个客户知识库设计教练：先诊断业务，再拆业务流，再配置能力开关，最后生成适合客户自己的目录、规则、模板和交付材料。

## 定位澄清

`client-kb-builder` 不是通用的知识库生成器，也不是“给任何资料自动建目录”的工具。

它更适合用在客户交付场景：当一个客户的业务需要被整理成可运行的知识库操作系统，并且需要同时考虑业务流、AI 协作边界、能力配置、交付培训和后续迭代时，这个 skill 才真正发挥价值。

如果只是做一个简单的个人资料库、读书笔记库或通用文档目录，它可能显得过重。此时应该使用轻量版输出，只保留诊断摘要、核心目录和少量模板。

## 它解决什么问题

很多客户说“我想做一个知识库”，但真实需求往往不是一个文件夹。

他们真正需要的是：

- 输入在哪里收集。
- 哪些信息值得沉淀。
- 业务怎么从线索、资料、项目、内容或课程推进到最终交付。
- 哪些环节可以让 AI 协助。
- 哪些动作必须人工确认。
- 交付物、复盘、案例和经验如何回流成资产。
- 客户自己以后怎么继续维护，而不是只拿到一堆目录。

`client-kb-builder` 解决的正是这个翻译问题：把客户业务翻译成知识库系统，把知识库系统翻译成可执行的 AI 协作规则。

## 适用场景

当你需要为客户设计以下内容时，可以使用这个 skill：

- 客户版知识库系统
- 企业或个人业务知识库操作系统
- AI 转型项目的知识库底座
- 内容团队、咨询团队、培训团队或项目团队的知识库架构
- 客户业务流、能力开关和目录结构
- 可交付的知识库搭建方案、培训方案和迭代清单
- 面向客户的 AI 执行规则、项目 skill 配置和验收清单

适合的客户类型包括：

- 内容/IP 型业务：选题、写作、发布、数据复盘和方法论复用。
- 咨询/服务型业务：客户诊断、方案生成、项目交付、案例沉淀和 SOP 升级。
- 课程/培训型业务：课程设计、教学资产、学员输入、答疑复用和班级复盘。
- 内部团队知识库：角色协作、项目资料、决策记录、交接文档和治理规则。
- AI 转型项目：场景发现、流程重构、能力配置、工具落地、培训和迭代证据。

## 核心原则

不要直接复制某个现有知识库的目录名或内部机制。

每一次客户知识库设计，都按这个顺序推进：

```text
五维诊断
-> 生命周期映射
-> 业务流包
-> 能力开关
-> 目录落位
-> 交付培训
-> 复盘迭代
```

如果客户需求还很模糊，先输出诊断清单和最小架构草案，不直接创建长期资产。

## 工作流程

1. 识别客户业务类型
   判断客户的收入模式、交付对象、输入来源、输出资产、协作角色、运行节奏和风险边界。

2. 进行五维诊断
   从战略上下文、输入系统、生产系统、交付分发、反馈治理五个维度理解客户业务。

3. 映射业务生命周期
   把客户真实业务翻译为输入、判断、处理、输出、复盘、沉淀、回流等阶段。

4. 设计业务流包
   每个业务流必须形成完整闭环，而不是单个任务、单个模板或单个目录。

5. 配置能力开关
   把 skill、Agent、CLI、模板、规则、自动化和外部工具都视为服务业务节点的能力。

6. 生成客户知识库目录
   使用客户自己的业务语言命名目录，并保持顶层目录按运行顺序排列。

7. 配置客户 AI 执行层
   明确 AI 能读什么、能写什么、能调用什么、能安装什么、何时必须停下来让人确认。

8. 输出交付材料
   包括搭建计划、客户说明、培训路径、首轮验收清单和下一版升级建议。

9. 记录复盘信号
   交付结束时记录使用卡点、缺失输入、待新增能力和后续产品化机会。

## 输出模式

轻量输出适合探索阶段，通常包括：

- 客户诊断摘要
- 推荐的知识库形态
- 第一版目录建议
- 缺失信息清单

完整交付输出适合落地阶段，通常包括：

- 客户五维诊断
- 生命周期映射表
- 业务流包清单
- 能力开关清单
- 客户知识库目录树
- AI 执行规则
- 系统级 skill 候选清单
- 项目级 skill 规划
- 关键模板建议
- 培训交付计划
- 复盘与升级清单

## 交付规模

为了避免对小客户过度设计，使用时应先判断交付规模。

### Lite 轻量版

适合个人、小团队、早期试点或需求还不清晰的客户。

输出重点：

- 一页客户诊断摘要
- 1 到 2 条核心业务流
- 简化目录树
- 少量高频模板
- 明确的下一步验证问题

### Standard 标准版

适合已经有稳定业务流、希望引入 AI 协作的客户。

输出重点：

- 五维诊断
- 生命周期映射
- 业务流开关表
- 能力开关表
- 客户知识库目录
- AI 执行边界
- 首批模板和交付说明

### Full Delivery 完整交付版

适合咨询交付、AI 转型、组织级知识库或需要长期维护的客户项目。

输出重点：

- 完整业务流包
- 系统级 skill 候选清单
- 项目级 skill 注册表
- 安装与启用计划
- 培训路径
- 验收记录
- 复盘和升级清单

## 能力边界

这个 skill 默认不写文件，除非用户明确要求保存、落地、生成文件或创建交付物。

客户知识库交付时需要注意：

- 不暴露交付方内部路径和同步机制。
- 不把内部客户名、私人上下文、未公开策略写进客户侧根目录。
- 不把系统级可复用 skill 和客户项目级 skill 混在一起。
- 自动化必须先归入业务流节点和能力开关，不能直接挂在目录上。
- 如果客户不需要完整系统，应主动建议更小版本。

## 安装与可用性说明

如果安装后暂时无法通过平台自动调用，可以先按手动方式使用：

1. 打开 `SKILL.md`，读取 Core Rule、Workflow、Output Modes 和 Boundaries。
2. 根据客户情况选择 Lite、Standard 或 Full Delivery。
3. 按需读取 `references/` 下的诊断、业务流、能力开关和验收模板。
4. 使用 `assets/customer-kb-directory-template.md` 作为目录起点，再按客户业务语言重命名。
5. 在最终交付前，用 `references/acceptance-checklist.md` 做一次检查。

如果用于参赛、公开展示或客户交付，建议明确标注当前版本和验证状态，避免让用户误以为它已经覆盖所有行业和所有平台环境。
