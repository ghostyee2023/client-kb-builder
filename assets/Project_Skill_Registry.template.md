 # 项目级 Skill 清单
 
 > 项目级 skill 是你业务专属的执行能力。每个 skill 必须服务某条业务流的某个节点，不能绕过业务流直接调用。
 > 
 > 三种交付类型：
 > - `delivered`：随知识库一起交付的，可在验证后启用。
 > - `custom`：你的业务专属定制，后续按实际流程创建。
 > - `training`：不直接交付成品，而是教你学会自己创建。

 ## 首版说明
 
 - 首版只登记 `planned` 状态，不默认启用。
 - 先跑通业务流，再根据实际卡点决定哪些节点需要封装成项目级 skill。
 - 连续跑通 3 次同一节点后，再考虑把它封装。
 - 项目级 skill 可以是 Codex skill、脚本、模板或纯人工 SOP，取决于你的工具环境。
 
 ## 创建指引
 
 当你准备好创建第一个项目级 skill 时：
 
 1. 确认这个 skill 服务哪条业务流的哪个节点。
 2. 定义它的读取范围（能看哪些目录）和写入范围（能写哪些目录）。
 3. 定义失败回退：如果 skill 执行失败，人工怎么做。
 4. 用 `skill-creator` 创建 skill 骨架。
 5. 在能力开关表登记，默认状态 `manual`。
 6. 跑一次验证，记录到 `Validation_Records/`。
