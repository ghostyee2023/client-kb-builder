 # 能力开关表
 
 > 能力开关控制"业务流中的某个节点是否调用具体能力"。每个能力必须绑定业务流节点，不能绕过业务流直接调用。
 > 
 > 状态说明：
 > - `core`：基础能力，常开，不能关闭。
 > - `on`：已启用，默认可用。
 > - `manual`：手动触发，需要你明确说"用"才会调用。
 > - `experimental`：试验中，仅供测试。
 > - `blocked`：缺依赖或权限，暂时不可用。

 ## 规则
 
 - `core` 能力不能关闭。
 - `manual` 能力必须由你明确触发。
 - AI 辅助能力不能直接生成最终对外交付物，必须经过你确认。
 - 任何自动化上线前，必须先定义读取范围、写入范围和失败回退。
 - 能力关闭时，只影响挂接它的节点，不直接关闭整条业务流。
 
 ## 新增能力登记
 
 当你需要新增一个能力（skill、模板、脚本、规则）时：
 
 1. 确认它服务哪条业务流的哪个节点。
 2. 在本表新增一行，状态先设为 `manual` 或 `experimental`。
 3. 定义读取范围、写入范围、失败回退。
 4. 跑一次验证，记录到 `Validation_Records/`。
 5. 验证通过后，把状态改为 `on`。
